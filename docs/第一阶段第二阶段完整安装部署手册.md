# HRClaw 第一阶段与第二阶段完整安装部署手册

## 1. 文档说明

本文档面向实施、运维和项目负责人，覆盖项目第一阶段与第二阶段的完整安装、部署、启动、升级、验收与运维流程。

适用范围：

- 第一阶段：BOSS 插件入库、即时打分、推荐处理台、Checklist、高级搜索、任务执行
- 第二阶段：`JD评分卡`、统一评分卡、批量简历导入、Word/PDF 解析、扫描件 OCR

项目根目录：

`/Users/jobs/Documents/CODEX/ZHAOPIN`

部署包目录：

`/Users/jobs/Documents/CODEX/ZHAOPIN/install`

## 2. 部署架构概览

本项目采用本地优先的单体部署方式：

- 后端：Python 标准库 HTTP 服务
- 数据库：SQLite
- 浏览器自动化：Playwright Chromium
- 后台页面：预构建 React + Tailwind 静态资源，由 Python 服务直接托管
- 插件：Chrome MV3 unpacked extension
- OCR：PaddleOCR

默认访问地址：

- 登录页：`http://127.0.0.1:8080/login`
- 默认账号：`admin / admin`
- 健康检查：`http://127.0.0.1:8080/health`

## 4. 服务器与软件要求

### 4.1 操作系统

- 推荐：macOS 或 Linux
- Chrome 插件使用端：Windows / macOS 均可，只要能访问部署后的服务地址

### 4.2 最低软件要求

- Python `>= 3.12`
- Node.js `>= 20`
- Chrome 浏览器
- 可访问外网以安装 Python / Node 依赖和 Playwright Chromium

### 4.3 目录与权限要求

部署账号至少需要具备以下权限：

- 对项目目录的读写权限
- 可创建 `.venv`
- 可写入 `data/`
- 可在 Chrome 中安装已解压插件

## 5. 部署前检查

正式安装前建议先确认：

1. 项目代码已经同步到目标机器
2. `install/` 目录存在且内容完整
3. `python3 --version` 输出不低于 `3.12`
4. `node -v` 与 `npm -v` 可正常执行
5. 目标端口 `8080` 未被其他服务长期占用

## 6. 第一阶段完整部署流程

第一阶段覆盖：

- Chrome 插件入库
- 推荐处理台
- Checklist
- 高级搜索
- 任务执行
- BOSS 会话同步与检查

### 6.1 安装 Python 运行环境

进入项目根目录后执行：

```bash
bash install/scripts/bootstrap_phase1_env.sh
```

该脚本会完成：

- 创建或复用 `.venv`
- 安装基础依赖
- 安装 Playwright Chromium
- 初始化运行目录

### 6.2 准备配置文件

本机部署：

```bash
cp install/packages/config/.env.local.example .env.local
```

内网部署：

```bash
cp install/packages/config/.env.intranet.example .env.local
```

建议重点检查以下配置：

```bash
SCREENING_SERVER_HOST=127.0.0.1
SCREENING_SERVER_PORT=8080
SCREENING_PUBLIC_BASE_URL=http://127.0.0.1:8080

SCREENING_BROWSER_AGENT=playwright
SCREENING_BROWSER_HEADLESS=false
SCREENING_BROWSER_STORAGE_STATE_PATH=/Users/jobs/Documents/CODEX/ZHAOPIN/data/auth/boss_storage_state.json

SCREENING_ENABLE_MODEL_EXTRACTION=true
SCREENING_EXTRACTION_PROVIDER=kimi_cli
SCREENING_EXTRACTION_MODEL=kimi-for-coding
SCREENING_KIMI_CLI_COMMAND=kimi
```

内网部署时通常需要改为：

```bash
SCREENING_SERVER_HOST=0.0.0.0
SCREENING_SERVER_PORT=8080
SCREENING_PUBLIC_BASE_URL=http://<内网IP>:8080
SCREENING_HEALTHCHECK_BASE_URL=http://127.0.0.1:8080
```

### 6.3 启动服务

```bash
bash install/scripts/start_server.sh
```

### 6.4 健康检查

```bash
bash install/scripts/check_health.sh
```

通过标准：

- `/health` 返回正常
- `/login` 可访问
- `data/screening.db` 已创建
- 插件目录完整

### 6.5 安装 Chrome 插件

插件目录：

`/Users/jobs/Documents/CODEX/ZHAOPIN/install/packages/chrome_extension/boss_resume_score`

安装步骤：

1. 打开 `chrome://extensions`
2. 打开“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择 `install/packages/chrome_extension/boss_resume_score`
5. 打开插件侧边栏
6. 在“后端地址”中填写：
   - 本机：`http://127.0.0.1:8080`
   - 内网：`http://<内网IP>:8080`

### 6.6 BOSS 登录与会话检查

当前流程不再由系统代替 HR 登录 BOSS，标准流程如下：

1. 在安装了插件的 Chrome 中打开 BOSS
2. HR 手动扫码登录
3. 登录后刷新一次 BOSS 页面
4. 打开平台 `任务执行`
5. 点击“检查已同步的 BOSS 会话”
6. 会话通过后再执行 Recommend 任务

### 6.7 第一阶段验收项

建议至少完成以下验收：

## 7. 第二阶段安装与升级流程

第二阶段在第一阶段基础上新增：

- `JD评分卡`
- 第一阶段与第二阶段评分卡统一管理
- JD 自动生成评分卡
- Word / PDF 批量简历导入
- 扫描件 PDF OCR fallback

### 7.1 第二阶段安装脚本

在第一阶段基础上执行：

```bash
bash install/scripts/bootstrap_phase2_env.sh
```

脚本默认会：

- 安装第二阶段文档解析依赖
- 尝试安装 OCR 依赖
- 继续安装 Playwright Chromium
- 初始化二阶段所需目录

如果你希望同时安装高级搜索增强依赖，可执行：

```bash
INSTALL_SEARCH_DEPS=1 bash install/scripts/bootstrap_phase2_env.sh
```

### 7.2 OCR 安装说明

第二阶段批量导入中的扫描件 PDF 识别依赖 PaddleOCR。

当前 `install/packages/python/wheels/` 已整理本机现成的：

- `paddlepaddle-3.3.0-cp312-cp312-macosx_11_0_arm64.whl`

说明：

- 该 wheel 适用于 `macOS arm64 + Python 3.12`
- 如果目标机器不是该平台，请按目标平台安装对应 PaddlePaddle 版本
- 安装 PaddlePaddle 成功后，再安装 `requirements-phase2-ocr.txt`

### 7.3 二阶段配置补充

如需启用 OCR，可在 `.env.local` 中增加或确认：

```bash
SCREENING_RESUME_OCR_PROVIDER=auto
SCREENING_RESUME_OCR_LANG=ch
SCREENING_RESUME_OCR_DET_MODEL_NAME=PP-OCRv5_mobile_det
SCREENING_RESUME_OCR_REC_MODEL_NAME=PP-OCRv5_mobile_rec
SCREENING_RESUME_OCR_FORCE_ON_PDF=false
```

如需更好的本地嵌入检索效果，可补充：

```bash
SCREENING_SEARCH_EMBEDDING_PROVIDER=auto
SCREENING_SEARCH_EMBEDDING_MODEL=BAAI/bge-base-zh-v1.5
SCREENING_SEARCH_EMBEDDING_DEVICE=cpu
```

### 7.4 二阶段后台验证

第二阶段部署完成后，建议重点验证：

1. `http://127.0.0.1:8080/hr/phase2` 页面可打开
2. 页面中能看到统一评分卡列表
3. 可以新建或编辑 `JD评分卡`
4. 批量导入 `pdf / doc / docx` 后能返回打分结果
5. 扫描版 PDF 在文本为空时能触发 OCR

## 11. 常见问题排查

### 11.1 服务启动后页面打不开

优先检查：

- `SCREENING_SERVER_PORT` 是否被占用
- `data/logs/phase1_server.log` 是否有异常
- `admin_frontend/dist` 是否存在

### 11.2 插件无结果或无法保存状态

优先检查：

- 插件侧边栏中的“后端地址”是否正确
- 当前是否位于候选人详情页
- 是否在 `chrome://extensions` 刷新过最新插件

### 11.3 BOSS 登录态检测失败

优先检查：

- HR 是否在安装了插件的 Chrome 中手动登录
- 登录后是否刷新过 BOSS 页面
- 平台中是否重新点击“检查已同步的 BOSS 会话”

### 11.4 扫描版 PDF 无法识别

优先检查：

- `paddlepaddle` 是否安装成功
- `paddleocr` 是否安装成功
- `.env.local` 中 OCR 配置是否正确
- 模型首次下载是否被网络限制

### 11.5 `JD评分卡` 页面打不开

优先检查：

- 后台静态资源是否存在
- `admin_frontend/dist/.vite/manifest.json` 是否存在
- 服务是否已重启到最新版本
