## 3. 环境准备

### 3.1 系统要求

- macOS 或 Linux
- Python `>= 3.12`
- Chrome 浏览器
- 本地可安装 Playwright Chromium

### 3.2 初始化环境

该脚本会执行：

- 创建运行所需目录

说明：

- `.env.local.example` 是安全模板，优先使用它
- 不要把真实密钥写回公开文档
- 项目会自动加载 `.env.local`

## 5. 首次保存 BOSS 登录态

### 5.1 保存登录态

```bash
PYTHONPATH=. .venv/bin/python scripts/save_boss_storage_state.py
```

脚本会：

1. 打开 BOSS 登录页
2. 由人工在浏览器中完成登录
3. 自动保存 Playwright `storage_state`

默认登录态文件：

`/Users/jobs/Documents/CODEX/ZHAOPIN/data/auth/boss_storage_state.json`

### 5.2 验证登录态

```bash
PYTHONPATH=. .venv/bin/python scripts/check_boss_storage_state.py
```

通过标准：

- 输出 `logged_in=true`
- 页面不再回到登录态

## 6. 启动与停止服务

### 6.1 后台启动

```bash
bash scripts/start_phase1_server.sh
```

默认行为：

- 按 `SCREENING_SERVER_HOST / SCREENING_SERVER_PORT` 绑定
- 日志写入 `data/logs/phase1_server.log`
- PID 写入 `data/pids/phase1_server.pid`
- 对外访问地址优先读取 `SCREENING_PUBLIC_BASE_URL`

### 6.2 停止服务

```bash
bash scripts/stop_phase1_server.sh
```

### 6.3 健康检查

```bash
bash scripts/check_phase1_health.sh
```

建议至少确认：

- `/health` 正常
- `/login` 可访问
- 插件目录存在且 `manifest.json` 存在

## 8. 插件标准使用流程

### 8.1 候选人详情页

当 HR 在 BOSS 打开候选人详情时：

1. 插件自动识别详情页
2. 自动执行候选人入库
3. 侧边栏秒级显示 `JD 速览`
4. 后端异步完成完整评分
5. HR 可直接保存跟踪状态

首次使用前，需要先在插件顶部配置后端地址：

- 本机后端：`http://127.0.0.1:8080`
- 内网服务器：`http://192.168.1.88:8080`

### 8.2 插件中可维护的跟踪状态

- 新入库
- 已评分
- 待复核
- 建议沟通
- 已沟通
- 待回复
- 待跟进
- 已邀约
- 面试已约
- 人才库
- 已淘汰
- 不再联系

保存状态后：

- 候选人状态会写入后台数据库
- `manual_stage_locked` 会开启
- 后续自动评分不会覆盖 HR 手工状态

### 8.3 插件字段说明

- `沟通结果`：记录 HR 最新联系结果
- `下次跟进时间`：用于待跟进场景
- `备注`：记录简历判断、沟通结论、复核信息

## 9. HR 后台使用

### 9.1 登录入口

- 登录页：`${SCREENING_PUBLIC_BASE_URL}/login`
- 默认账号：`admin / admin`

### 9.2 推荐处理台

地址：

`${SCREENING_PUBLIC_BASE_URL}/hr/workbench`

支持能力：

- 按岗位、来源、阶段、人工接管状态筛选
- 查看候选人分数、原因、快照文本
- 查看状态流转和 timeline
- 与插件保存状态结果联动

推荐关注的筛选项：

- `来源 = 插件入库`
- `人工状态锁 = 是`

### 9.3 其他后台页面

- Checklist：`${SCREENING_PUBLIC_BASE_URL}/hr/checklist`
- 高级搜索：`${SCREENING_PUBLIC_BASE_URL}/hr/search`
- 任务页：`${SCREENING_PUBLIC_BASE_URL}/hr/tasks`

## 13. 常见问题

### 13.1 服务无法启动

- 确认端口 `8080` 未被占用
- 查看日志：`tail -f data/logs/phase1_server.log`

### 13.2 插件不识别详情页

- 确认当前是候选人详情，不是推荐列表页
- 在 `chrome://extensions` 刷新插件
- 确认服务已启动
- 确认插件中的“后端地址”已改成真实服务地址

### 13.3 插件保存状态后看起来没变

- 确认扩展已经刷新到最新版本
- 再次打开候选人详情页，检查 `候选人 ID` 是否已显示
- 到 Workbench 中确认该候选人的状态是否已更新

### 13.4 Workbench 队列姓名异常

已知历史脏数据可能来自更早的采集版本。当前版本已经拦截：

- `/web/chat/recommend`
- PDF 预览页
- `JD 速览 / 职位管理 / 个人资料` 这类非候选人详情内容

### 13.5 BOSS 再次要求登录

- 重新执行 `scripts/save_boss_storage_state.py`
- 确认 `.env.local` 中的 `SCREENING_BROWSER_STORAGE_STATE_PATH` 指向正确文件
