# HRClaw 内网部署实施清单

## 1. 文档用途

本文档用于指导项目在公司内网环境完成从“服务器准备”到“正式上线”的实施落地，适用当前单机内网部署方案。

适用对象：

- 运维
- IT 支持
- 项目负责人
- 招聘运营负责人

项目根目录：

`/Users/jobs/Documents/CODEX/ZHAOPIN`

部署包目录：

`/Users/jobs/Documents/CODEX/ZHAOPIN/install`

相关主手册：

- `docs/第一阶段第二阶段完整安装部署手册.md`
- `docs/第二阶段平台操作说明书.md`

## 2. 上线前统一约定

在实施前先明确以下事项：

- [ ] 确认部署方式为“1 台内网应用服务器”
- [ ] 确认服务访问方式：`http://内网IP:8080` 或 `https://内网域名`
- [ ] 确认上线范围：第一阶段 + 第二阶段全部启用
- [ ] 确认是否启用扫描版 PDF OCR
- [ ] 确认是否由 `1 个指定 HR/运营账号` 负责 Recommend 自动化任务
- [ ] 确认 10 位 HR 的插件安装方式：
  - 手动安装 unpacked extension
  - 或由 IT 统一下发
- [ ] 确认备份落点：NAS 或内网备份服务器

## 3. 角色分工

### 3.1 运维

- [ ] 准备服务器
- [ ] 安装运行环境
- [ ] 部署应用
- [ ] 配置反向代理
- [ ] 建立备份与巡检

### 3.2 IT 支持

- [ ] 协助 Chrome 插件在 10 台 HR 电脑安装
- [ ] 放通内网访问地址和端口
- [ ] 如有 HTTPS，分发公司证书

### 3.3 招聘运营负责人

- [ ] 指定 Recommend 自动化值班账号
- [ ] 验收 JD评分卡、简历导入、Workbench、Checklist
- [ ] 组织 HR 培训

## 4. 服务器准备清单

### 4.1 服务器资源

- [ ] 操作系统：`Ubuntu 22.04 LTS`
- [ ] CPU：不少于 `8 vCPU`
- [ ] 内存：不少于 `16 GB`
- [ ] 磁盘：不少于 `300 GB SSD`
- [ ] 内网固定 IP 已分配
- [ ] 允许 SSH 登录

正式推荐值：

- [ ] `16 vCPU / 32 GB RAM / 500 GB SSD`

### 4.2 系统基础软件

- [ ] `python3 --version` >= `3.12`
- [ ] `node -v` >= `20`
- [ ] `npm -v` 可正常执行
- [ ] `google-chrome` 或 `chromium` 可安装
- [ ] `unzip`、`tar`、`curl` 可正常执行

### 4.3 系统账号与目录

- [ ] 创建运行账号，例如 `screening`
- [ ] 项目目录具备读写权限
- [ ] `data/` 目录具备读写权限
- [ ] `.env.local` 仅运维可读写

## 5. 网络与访问准备

### 5.1 端口放通

- [ ] HR 网段 -> 应用入口 `80/443`
- [ ] 运维网段 -> SSH `22`
- [ ] Python 服务端口 `8080` 仅本机或反向代理可访问

### 5.2 域名与证书

- [ ] 确认是否使用内网域名
- [ ] 如使用 HTTPS，证书已签发
- [ ] 如使用内部 CA，10 台 HR 电脑已安装根证书

## 6. 部署包检查

进入项目目录后，确认以下内容存在：

- [ ] `install/README.md`
- [ ] `install/packages/python/requirements-phase1.txt`
- [ ] `install/packages/python/requirements-phase2-ocr.txt`
- [ ] `install/packages/config/.env.local.example`
- [ ] `install/packages/config/.env.intranet.example`
- [ ] `install/packages/frontend/admin_frontend-dist.tgz`
- [ ] `install/packages/chrome_extension/boss_resume_score`
- [ ] `install/scripts/bootstrap_phase1_env.sh`
- [ ] `install/scripts/bootstrap_phase2_env.sh`
- [ ] `install/scripts/start_server.sh`
- [ ] `install/scripts/check_health.sh`

## 7. 第一阶段安装实施

### 7.1 安装基础环境

- [ ] 执行：

```bash
bash install/scripts/bootstrap_phase1_env.sh
```

- [ ] 确认 `.venv` 已创建
- [ ] 确认 Playwright Chromium 已安装
- [ ] 确认 `data/auth data/logs data/pids data/resumes data/qdrant_search` 已创建

### 7.2 生成配置文件

- [ ] 复制内网模板：

```bash
cp install/packages/config/.env.intranet.example .env.local
```

- [ ] 修改 `SCREENING_PUBLIC_BASE_URL`
- [ ] 修改为真实内网 IP 或域名
- [ ] 检查 `SCREENING_SERVER_HOST=0.0.0.0`
- [ ] 检查 `SCREENING_SERVER_PORT=8080`

### 7.3 启动应用

- [ ] 执行：

```bash
bash install/scripts/start_server.sh
```

- [ ] 确认日志文件已生成：`data/logs/phase1_server.log`
- [ ] 确认 PID 文件已生成：`data/pids/phase1_server.pid`

### 7.4 健康检查

- [ ] 执行：

```bash
bash install/scripts/check_health.sh
```

- [ ] 浏览器访问：
  - `http://<内网IP>:8080/health`
  - `http://<内网IP>:8080/login`

## 8. 第二阶段安装实施

### 8.1 安装二阶段能力

- [ ] 执行：

```bash
bash install/scripts/bootstrap_phase2_env.sh
```

- [ ] 确认 `pypdf` 已可用
- [ ] 确认 OCR 依赖已安装或已按平台准备

### 8.2 OCR 检查

如启用扫描件 PDF OCR：

- [ ] 确认 `paddlepaddle` 已安装
- [ ] 确认 `paddleocr` 已安装
- [ ] `.env.local` 中 OCR 配置已确认
- [ ] 首次模型下载网络可用

### 8.3 二阶段页面检查

- [ ] 访问 `http://<内网IP>:8080/hr/phase2`
- [ ] 确认页面可打开
- [ ] 确认能看到统一评分卡列表
- [ ] 确认可进入批量导入区域

## 9. 反向代理与常驻运行

### 9.1 Nginx

- [ ] 配置内网入口域名或 IP
- [ ] 反向代理到 `127.0.0.1:8080`
- [ ] 如使用 HTTPS，证书配置完成

建议规则：

- [ ] `/` -> `http://127.0.0.1:8080`
- [ ] 保留真实客户端 IP
- [ ] 设置合理超时时间

### 9.2 systemd

- [ ] 建立 `screening.service`
- [ ] 设置开机自启
- [ ] 设置异常自动拉起
- [ ] `WorkingDirectory` 指向项目根目录
- [ ] `User` 指向运行账号

## 10. HR 电脑实施清单

针对 10 位 HR，逐台确认：

- [ ] 已安装 Chrome 浏览器
- [ ] 可访问内网部署地址
- [ ] 已安装 `boss_resume_score` 插件
- [ ] 插件中“后端地址”已配置为真实内网地址
- [ ] 如使用 HTTPS 私有 CA，证书已安装
- [ ] 可正常打开插件侧边栏

说明：

- HR 电脑不需要安装 Python
- HR 电脑不需要安装 Node.js
- HR 电脑不需要安装 PaddleOCR
- HR 电脑不需要安装数据库

## 11. BOSS 使用流程实施

- [ ] 指定 Recommend 自动化值班账号
- [ ] 指定负责同步 BOSS 会话的 HR
- [ ] 在该 HR 的 Chrome 中手动登录 BOSS
- [ ] 登录后刷新一次 BOSS 页面
- [ ] 回到平台点击“检查已同步的 BOSS 会话”
- [ ] 会话通过后再执行 Recommend 任务

说明：

- 当前 Recommend 自动化建议由 `1 个账号` 统一执行
- 10 位 HR 可以同时使用后台、插件、JD评分卡和 Workbench
- 不建议 10 人同时各自跑 Recommend 自动化任务

- [ ] 插件评分正常回填
- [ ] Workbench 可见候选人记录

### 12.2 并发验收

- [ ] 至少 3 台 HR 电脑同时使用插件无异常

## 14. 备份与回滚

### 14.1 每日备份

- [ ] 备份 `data/screening.db`
- [ ] 备份 `data/auth/`
- [ ] 备份 `data/resumes/`
- [ ] 备份 `.env.local`

### 14.2 升级前备份

- [ ] 手工打包 `data/`
- [ ] 备份当前 `.env.local`
- [ ] 记录当前版本和上线时间

### 14.3 回滚策略

如果上线后出现严重问题：

- [ ] 停止当前服务
- [ ] 恢复上一版代码
- [ ] 恢复上一版 `data/` 备份
- [ ] 恢复上一版 `.env.local`
- [ ] 重启服务
- [ ] 重新验证 `/health` 和 `/login`

- [ ] 检查 CPU 和内存是否持续过高
- [ ] 检查磁盘增长速度
- [ ] 检查 OCR 是否频繁失败
- [ ] 检查 Workbench 是否存在异常脏数据
- [ ] 检查插件是否有大面积无法连接后端

- [ ] 判断是否需要下一步做多用户 BOSS 会话隔离

- [ ] `install/` 目录
